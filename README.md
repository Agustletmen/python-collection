TCP 是流式协议（像水管里的水流），不是 “数据包” 协议：
粘包：客户端连续发A和B，服务器可能一次收到AB（数据粘在一起）。
拆包：客户端发大数据包ABCDE，服务器可能先收AB，再收CDE（数据被拆分）。
产生原因：TCP 的 Nagle 算法、滑动窗口、MTU（最大传输单元）限制。

一个特征基本等价于一个维度
一、先把核心概念讲透：特征 = 维度，分类 = 多维空间的 “划分”
1. 单个特征（单维度）的分类
比如你要判断 “水果是否是苹果”，只看一个特征（维度）：“颜色是否为红色”。
维度：颜色（取值：红 / 绿 / 黄）；
分类逻辑：红色→苹果，非红色→不是苹果（这是单维度的简单分类）。
但这个判断很不准（比如红石榴也会被误判），因为只用了 1 个维度。
2. 多个特征（多维度）的分类
还是判断 “是否是苹果”，用 3 个特征（3 个维度）：
维度 1：颜色（红 / 绿）；
维度 2：形状（圆形 / 椭圆形）；
维度 3：口感（脆 / 面）。
模型会学习这 3 个维度的组合规律：
颜色 = 红 + 形状 = 圆形 + 口感 = 脆 → 是苹果；
颜色 = 绿 + 形状 = 圆形 + 口感 = 脆 → 是青苹果；
颜色 = 红 + 形状 = 椭圆形 + 口感 = 面 → 不是苹果（比如红李子）。
👉 关键：不是 “多个维度各自分类后叠加结果”，而是 “多个维度共同构成一个‘特征空间’，模型在这个空间里找到划分不同类别的边界”。


二、结合 tsfresh 特征的实际例子（时序分类）
比如你要做 “设备是否故障” 的二分类：
用 tsfresh 提取 3 个特征（3 个维度）：
维度 1：传感器数值的均值；
维度 2：传感器数值的峰值数量；
维度 3：传感器序列的趋势斜率。
正常设备的特征组合：均值 = 50 + 峰值数 = 2 + 斜率 = 0.1；
故障设备的特征组合：均值 = 80 + 峰值数 = 10 + 斜率 =-0.5；
模型（比如随机森林）会学习 “这 3 个维度的数值组合” 和 “故障 / 正常” 的对应关系，最终根据新设备的 3 个维度数值，判断它属于哪一类。

很多机器学习模型就是把概率模型套上了数据训练流程


概率模型是基于概率论和统计学的框架，用于描述数据生成过程或对未知变量进行推断。其核心思想是通过概率分布表示变量间的关系，并利用概率规则（如贝叶斯定理、最大似然估计）进行预测、分类或生成。
生成模型是建模联合概率分布 (P(X, Y)) 的模型，能够生成新数据样本。常见的生成模型包括朴素贝叶斯、高斯混合模型（GMM）、隐马尔可夫模型（HMM）和生成对抗网络（GAN）。这些模型广泛应用于文本生成、图像合成和语音识别等领域。
判别模型直接建模条件概率 (P(Y|X)) 或决策边界，常见的模型有逻辑回归、条件随机场（CRF）和支持向量机（SVM）。它们主要用于分类任务（如垃圾邮件检测）和序列标注（如词性标注）。
概率图模型通过图结构表示联合概率分布，节点表示变量，边表示依赖关系。贝叶斯网络（有向无环图）和马尔可夫随机场（无向图）是两种主要类型，分别用于因果推理和变量关联建模。
常见的概率分布模型包括离散分布（如伯努利分布、二项分布、泊松分布）和连续分布（如正态分布、指数分布、贝塔分布）。例如，正态分布是最常见的连续分布，其概率密度函数由均值和方差决定。
