一个特征基本等价于一个维度
一、先把核心概念讲透：特征 = 维度，分类 = 多维空间的 “划分”
1. 单个特征（单维度）的分类
   比如你要判断 “水果是否是苹果”，只看一个特征（维度）：“颜色是否为红色”。
   维度：颜色（取值：红 / 绿 / 黄）；
   分类逻辑：红色→苹果，非红色→不是苹果（这是单维度的简单分类）。
   但这个判断很不准（比如红石榴也会被误判），因为只用了 1 个维度。
2. 多个特征（多维度）的分类
   还是判断 “是否是苹果”，用 3 个特征（3 个维度）：
   维度 1：颜色（红 / 绿）；
   维度 2：形状（圆形 / 椭圆形）；
   维度 3：口感（脆 / 面）。
   模型会学习这 3 个维度的组合规律：
   颜色 = 红 + 形状 = 圆形 + 口感 = 脆 → 是苹果；
   颜色 = 绿 + 形状 = 圆形 + 口感 = 脆 → 是青苹果；
   颜色 = 红 + 形状 = 椭圆形 + 口感 = 面 → 不是苹果（比如红李子）。
   👉 关键：不是 “多个维度各自分类后叠加结果”，而是 “多个维度共同构成一个‘特征空间’，模型在这个空间里找到划分不同类别的边界”。


二、结合 tsfresh 特征的实际例子（时序分类）
比如你要做 “设备是否故障” 的二分类：
用 tsfresh 提取 3 个特征（3 个维度）：
维度 1：传感器数值的均值；
维度 2：传感器数值的峰值数量；
维度 3：传感器序列的趋势斜率。
正常设备的特征组合：均值 = 50 + 峰值数 = 2 + 斜率 = 0.1；
故障设备的特征组合：均值 = 80 + 峰值数 = 10 + 斜率 =-0.5；
模型（比如随机森林）会学习 “这 3 个维度的数值组合” 和 “故障 / 正常” 的对应关系，最终根据新设备的 3 个维度数值，判断它属于哪一类。



tsfresh：自动化时序特征提取（无需手动构造特征）；
物联网时序数据的挖掘效果，80% 取决于特征工程，但手动构造特征（如偏度、峰度、傅里叶变换、熵值等）既耗时又易遗漏
tsfresh 可自动从时序数据中提取数百种专业时序特征，支持均值、方差、极值、趋势、周期性、熵值、小波变换等上千种特征
Time Series Feature extraction based on scalable hypothesis tests

AutoRegressive Integrated Moving Average


fbprophet：Facebook 开源的时序预测库（适配缺失值、异常值）。
